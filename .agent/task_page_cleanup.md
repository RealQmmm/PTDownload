# ✅ RSS 任务界面简化 - 移除内嵌路径管理

## 变更说明

移除了 RSS 任务创建/编辑界面中的路径管理功能，因为系统已提供了独立的"路径管理"页面。

## 修改内容

### 移出的功能
1.  **路径管理入口**：移除了 RSS 任务表单中路径选择框旁的齿轮图标按钮。
2.  **路径管理模态框**：移除了页面底部的路径增删改查模态框。
3.  **冗余代码**：清理了相关的 React 状态（`showPathsModal`, `pathFormData` 等）和提交/删除处理函数。

### 保留的功能
*   用户仍然可以在 RSS 任务表单中选择已有的下载路径。
*   "自定义路径"选项仍然保留，允许为特定任务手动输入一次性路径。
*   页面仍然会通过 `fetchDownloadPaths` 加载路径列表以供选择。

## 目的
*   **简化界面**：减少 RSS 任务配置时的视觉干扰。
*   **职责分离**：任务管理归任务管理，路径配置归设置/路径页面，避免功能重叠。
*   **代码清理**：减少 `TasksPage.jsx` 的代码量，提高可维护性。
