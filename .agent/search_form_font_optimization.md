# 搜索框移动端字体优化

## 修改时间
2026-01-03 22:43

## 问题描述

**现象**: 在手机上，"搜索"两个字换行了，因为宽度不够

**原因**: 
1. 字体太大（默认 14px）
2. 站点选择宽度过宽（112px）
3. 搜索按钮宽度过宽（64px）
4. 元素间距过大（8px）

---

## 解决方案

### 1. 减少字体大小

#### 搜索框
```jsx
// 修改前
<Input placeholder="输入关键词..." />

// 修改后
<Input 
    placeholder="输入关键词..." 
    className="text-xs sm:text-sm"  // 小屏幕 12px，中等屏幕 14px
/>
```

#### 站点选择
```jsx
// 修改前
<Select>...</Select>

// 修改后
<Select className="text-xs sm:text-sm">  // 小屏幕 12px，中等屏幕 14px
    ...
</Select>
```

#### 搜索按钮
```jsx
// 修改前
<Button className="w-16 sm:w-20 shrink-0">搜索</Button>

// 修改后
<Button className="w-14 sm:w-20 shrink-0 text-xs sm:text-sm px-2 sm:px-4">
    搜索
</Button>
```

---

### 2. 优化宽度分配

#### 元素间距
```jsx
// 修改前
gap-2  // 8px

// 修改后
gap-1.5 sm:gap-2  // 小屏幕 6px，中等屏幕 8px
```

#### 站点选择宽度
```jsx
// 修改前
w-28 sm:w-32  // 小屏幕 112px，中等屏幕 128px

// 修改后
w-24 sm:w-32  // 小屏幕 96px，中等屏幕 128px
```

#### 搜索按钮宽度
```jsx
// 修改前
w-16 sm:w-20  // 小屏幕 64px，中等屏幕 80px

// 修改后
w-14 sm:w-20  // 小屏幕 56px，中等屏幕 80px
```

#### 搜索按钮内边距
```jsx
// 新增
px-2 sm:px-4  // 小屏幕 8px，中等屏幕 16px
```

### 修改后（小屏幕 < 375px）：

**宽度计算**:

### 3. 响应式优化 📱
- 小屏幕更紧凑
- 中等屏幕更舒适
- 平滑过渡

### 4. 空间利用 📊
- 搜索框获得更多空间
- 所有元素在同一行
- 提高输入效率
